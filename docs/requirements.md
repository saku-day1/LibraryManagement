# 要件定義

## システム概要

| 項目 | 内容 |
|---|---|
| システム名 | LibraryManagement |
| 目的 | 書籍の読書ステータスをカンバン形式で視覚的に管理する |
| 対象ユーザー | 個人利用 |

## 機能要件

### カンバンボード

- 未読 / 読書中 / 読了 の3カラムを横並びで表示
- 各カラムに属する書籍カードを一覧表示
- カードのドラッグ&ドロップでステータスを変更
- ステータス変更は即時 API へ反映

### 書籍登録

| フィールド | 必須 | 備考 |
|---|---|---|
| タイトル | ✓ | 最大 255 文字 |
| 著者 | ✓ | 最大 100 文字 |
| ステータス | ✓ | unread / reading / completed |
| ISBN | - | 13 桁 |
| 表紙画像 URL | - | 外部 URL |

### 書籍詳細・編集

| フィールド | 必須 | 備考 |
|---|---|---|
| 評価 | - | 1〜5 の星評価（読了時のみ） |
| 感想メモ | - | 自由記述 |
| 読み始め日 | - | |
| 読了日 | - | |

### 書籍削除

- カードから削除操作が可能
- 削除前に確認ダイアログを表示

## 非機能要件

| 項目 | 要件 |
|---|---|
| レスポンス | API レスポンス 200ms 以内（LAN） |
| セキュリティ | CORS 設定・SQL インジェクション対策 |
| コスト | EC2 t3.micro + RDS db.t3.micro の最小構成 |
| 可用性 | シングルインスタンス（課題用途） |

## 非スコープ

以下の機能は今回の実装対象外とする。

| 機能 | 理由 |
|---|---|
| ユーザー認証・ログイン | 個人利用のためアカウント管理は不要 |
| 複数ユーザー対応 | 単一ユーザーの利用を前提とする |
| 書籍検索・フィルタリング | MVP としてはカンバン表示のみで十分 |
| 外部 API 連携（Amazon・楽天など） | ISBN からの書籍情報自動取得は対象外 |
| 読書進捗（ページ数・%） | ステータス管理のみで十分 |
| タグ・カテゴリ管理 | シンプルな構成を優先 |
| 通知・リマインダー機能 | 課題スコープ外 |
| モバイルアプリ（iOS/Android） | Web ブラウザでの利用のみ対応 |
| データエクスポート（CSV など） | 課題スコープ外 |
