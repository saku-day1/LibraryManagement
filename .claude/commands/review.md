現在のブランチの実装が、要件定義・詳細設計・フェーズの完了条件に沿っているかをレビューしてください。

## 手順

1. `docs/requirements.md` を読み、機能要件・非機能要件を把握する
2. `docs/detailed-design.md` を読み、該当フェーズの「完了条件」「作成・編集するファイル」「API設計」「DB設計」を把握する
3. `git diff main` で現在の変更差分を確認する
4. 以下のチェック項目を順番に確認し、結果を報告する

## チェック項目

### 要件との整合性
- [ ] 実装した機能が要件定義の機能要件を満たしているか
- [ ] 要件にない機能が勝手に追加されていないか
- [ ] フィールド名・型・制約が DB 設計と一致しているか

### 詳細設計との整合性
- [ ] ファイルパスとコンポーネント名が詳細設計のディレクトリ構成と一致しているか
- [ ] API のエンドポイント・メソッド・レスポンス形式が API 設計と一致しているか
- [ ] 使用している技術・ライブラリが技術スタックに沿っているか（Vue 3 + TypeScript / Rails 7 / MySQL 8.0）

### フェーズ完了条件
- [ ] 詳細設計に記載された当該フェーズの完了条件をすべて満たしているか
- [ ] 次フェーズの作業が混入していないか（スコープ超過がないか）

## 報告フォーマット

問題がない場合:
「要件定義・詳細設計との整合性に問題はありません。フェーズ N の完了条件をすべて満たしています。」

問題がある場合:
| 種別 | 箇所 | 内容 | 対応 |
|------|------|------|------|
| 不一致 | ファイル名:行番号 | 設計では X だが Y になっている | Z に修正する |
| 不足 | - | 完了条件「〇〇」が未実装 | 実装を追加する |
| 超過 | ファイル名 | フェーズ N+1 の作業が含まれている | 別ブランチに分離する |
