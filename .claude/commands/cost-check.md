実装コストおよび技術的負債の観点で現在のブランチの変更差分を確認してください。以下の項目をチェックし、懸念点があれば具体的に報告してください。

## チェック項目

1. **依存関係の重さ**
   - 追加した gem / npm パッケージのサイズと必要性を確認
   - 同等の機能が既存ライブラリや標準ライブラリで実現できないか

2. **パフォーマンス**
   - N+1 クエリが発生していないか (Rails: `includes` の使用確認)
   - 不必要なループや再計算がないか
   - フロントエンドで不要な再レンダリングが発生しないか (Vue の `computed` / `watch` の適切な使用)

3. **実装の複雑さ**
   - 過度な抽象化や早すぎる汎化がないか
   - 現在の要件に不要な機能が含まれていないか

4. **保守性**
   - 重複コードがないか
   - 命名が意図を適切に表しているか

5. **AWS コスト (インフラ変更がある場合)**
   - EC2 / RDS インスタンスタイプが t3.micro の範囲内か
   - 不要なリソースが作成されていないか

## 報告フォーマット

- 問題なし: 「実装コスト上の懸念はありません」と報告
- 懸念あり: 影響度 (High / Medium / Low) と改善提案を明示
