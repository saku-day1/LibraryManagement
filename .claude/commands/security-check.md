セキュリティ観点で現在のブランチの変更差分を確認してください。以下の項目を必ずチェックし、問題があれば具体的なファイル名と行番号を示して報告してください。

## チェック項目

1. **機密情報の漏洩**
   - API キー、パスワード、トークン、シークレットがコードにハードコードされていないか
   - `.env` ファイルや認証情報が git 追跡対象になっていないか

2. **インジェクション対策**
   - SQL インジェクション: ActiveRecord の生クエリ使用箇所の確認
   - コマンドインジェクション: `system()` や `` ` `` バッククォートの使用確認

3. **CORS 設定**
   - Rails の `rack-cors` 設定で不必要にワイルドカード (`*`) を許可していないか

4. **認証・認可**
   - 認証なしで機密データにアクセスできるエンドポイントがないか

5. **依存関係の脆弱性**
   - `bundle audit` (Ruby) / `npm audit` (Node) の結果確認

## 報告フォーマット

- 問題なし: 「セキュリティ上の問題は検出されませんでした」と報告
- 問題あり: 重大度 (High / Medium / Low) と修正方法を明示
